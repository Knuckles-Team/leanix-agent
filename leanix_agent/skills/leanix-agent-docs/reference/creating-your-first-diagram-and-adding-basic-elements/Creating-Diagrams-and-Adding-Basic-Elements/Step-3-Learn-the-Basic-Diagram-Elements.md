##  Step 3: Learn the Basic Diagram Elements
Element | Description | Usage Tips
---|---|---
Shapes | Visual components used to represent architectural elements like applications, capabilities, or processes. |
  * Drag shapes from the shape library or use the toolbar.
  * Shapes can be linked to fact sheets or used as placeholders.


Connectors | Lines or arrows that define relations or data flow between shapes. |
  * Use connectors to establish relations between fact sheets.
  * Right-click to create new relations and sync with the inventory.


Fact sheets | Inventory items that can be inserted into diagrams as shapes |
  * Insert from toolbar or right-click.
  * Can link, unlink, or create new fact sheets.


Text and labels | Descriptive text added to shapes or connectors to clarify meaning. |
  * Apply predefined labels to fact sheets.
  * Custom text can be added manually.


Colors and styles | Visual styling options to differentiate elements or reflect inventory attributes. |
  * Colors may automatically update based on changes in the inventory.
  * Use the format panel to adjust manually.


Layers | Organizational tool to separate diagram content into abstraction levels. |
  * Add layers from the top menu.
  * Toggle visibility to reduce clutter and focus on specific architecture views.


Views | Color-coded overlays that represent attributes or relations from inventory. |
  * Apply views from the toolbar.
  * Saved views persist with the diagram.


Interfaces | Represent data exchange between applications in data flow diagrams. | Use to model integration scenarios and visualize protocols and data exchanged.
SAP-specific shapes  | Specialized shapes for SAP BTP reference architecture modeling. |
  * Available in the SAP-specific shape library.
  * Use for standardized visualizations.
