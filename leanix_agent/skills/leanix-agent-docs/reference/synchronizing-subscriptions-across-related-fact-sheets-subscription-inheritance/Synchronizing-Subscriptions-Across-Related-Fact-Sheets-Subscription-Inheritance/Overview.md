##  Overview
Subscription inheritance ensures that ownership and responsibilities are automatically kept in sync across related fact sheets. For example, when an application owner is assigned to an application, they should typically also be responsible for the underlying IT components that support that application. By using automations for this scenario, you can:
  * Reduce manual updates: Eliminate the need to manually add or remove subscriptions on related fact sheets.
  * Improve data consistency: Ensure that ownership information is always up to date across the architecture.
  * Maintain accuracy: Automatically assign owners when new relations are created. Remove outdated subscriptions when relations are removed or ownership changes.
