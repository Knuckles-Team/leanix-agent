##  Best Practices
  * Use diagram templates for standardized, data-driven structures, not for customization. If you need to add custom shapes or additional labels, you can start from a template, save it as a static diagram, and then add custom shapes.
  * Design templates so that the preview is useful on its own. Most users will have view-only access and will rely on the real-time preview to understand the current inventory state.
  * Choose a default view and label setup in the template editor that reflects how you want to interpret the diagram, such as by lifecycle or obsolescence risk. This ensures that diagrams rendered from the template are immediately meaningful.
  * Review your templates regularly. Retire outdated ones to keep the template library clear and manageable.
