##  Preset Diagram Templates
When you create a new diagram template, you can start from scratch or use an out-of-the-box presets. The preset templates include predefined fact sheet types, relations, and views for common scenarios.
Template | Example | When to Use
---|---|---
Application context |  ![Template showing an application linked to a business capability in SAP LeanIX.](https://help.sap.com/doc/72d375467c1e4dcb872dfa2998b6328d/CLOUD/en-US/loioe86aac6abeec4c729cdc29f16fc4809d_LowRes.png) A central application node that contains its organization (owning team), connected with an arrow to a related business capability. |
  * To quickly identify who owns an application and which business capabilities it supports.
  * Best for stakeholder alignment, scoping workshops, and impact analysis.


Application technology architecture |  ![Template showing data flow from application to IT component in SAP LeanIX.](https://help.sap.com/doc/72d375467c1e4dcb872dfa2998b6328d/CLOUD/en-US/loioe7dd505ccf324dde8c29f7475bf2a45e_LowRes.png) A one-directional visualization showing a single application with a dependent IT component. The template highlights the technical‑fit classification. |
  * To assess the technical dependencies of an application and understand the suitability or risk level of the underlying IT components.
  * Best for architecture assessments, technical debt reviews, or upgrade planning.
