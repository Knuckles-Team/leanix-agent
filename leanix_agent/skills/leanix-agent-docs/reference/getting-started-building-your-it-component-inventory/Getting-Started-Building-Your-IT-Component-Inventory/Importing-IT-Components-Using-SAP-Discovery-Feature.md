##  Importing IT Components Using SAP Discovery Feature
The SAP discovery feature discovers SAP services and systems, helping you build a comprehensive view of the SAP landscape in your inventory. Through integrations with SAP Cloud ALM and SAP Build, it discovers the following:
  * SaaS products and SaaS ERP entries
  * SAP Private Cloud products and SAP Private Cloud ERP entries
  * SAP On-Premise products, including ERP systems.
  * Custom-built extensions from SAP Build projects, including apps, code, and process automation.


The discovered items, including IT components, are listed in the SAP discovery inbox. In the inbox, you can do the following:
  * Review the discovered items and create corresponding fact sheets.
  * If a corresponding IT component fact sheet already exists in your inventory, then you can link the item to the existing fact sheet.
  * Create relations to other fact sheets that are discovered together with the main service or system.


At the same time, the fact sheets are also linked to the reference catalog item, ensuring they stay updated as the catalog data evolves. It also links the IT component fact sheet to the associated provider and tech category fact sheets if you have configured the reference catalog that way. To learn more, see [IT Components in the Reference Catalog](https://help.sap.com/docs/leanix/ea/it-components-in-reference-catalog?locale=en-US&state=PRODUCTION&version=CLOUD "By using reference data for IT component fact sheets, you can increase standardization and data quality, set relations to other fact sheets for best-practice modeling, as well as manage technology obsolescence risks by using vendor lifecycle information.").
For a more detailed guide, see [SAP Discovery](https://help.sap.com/docs/leanix/ea/sap-discovery?locale=en-US&state=PRODUCTION&version=CLOUD "Identify and manage your organization's SAP systems, services, and custom-built extensions").
