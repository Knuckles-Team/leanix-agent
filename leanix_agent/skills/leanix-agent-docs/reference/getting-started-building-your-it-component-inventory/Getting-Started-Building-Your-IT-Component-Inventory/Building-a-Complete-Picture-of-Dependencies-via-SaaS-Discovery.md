##  Building a Complete Picture of Dependencies via SaaS Discovery
SaaS discovery identifies your organization's Software as a Service (SaaS) applications through integrations with third-party systems. Although it doesn't discover and import IT components, it links associated IT components with the discovered SaaS. For a detailed guide, see [SaaS Discovery](https://help.sap.com/docs/leanix/ea/saas-discovery?locale=en-US&state=PRODUCTION&version=CLOUD "SaaS discovery identifies your organization's SaaS applications by integrating with SSO, SASE, and CASB solutions. It then updates and enriches fact sheets using the reference catalog.").
Discovered SaaS are listed in the SaaS discovery inbox. You can review the discovered SaaS items and create corresponding fact sheets or link the item to the existing application fact sheets.
At the same time, the fact sheets are also linked to the reference catalog item. This also creates corresponding links to underlying IT components, as well as associated provider and technology category fact sheets. Creating a complete view of IT components and dependencies. To learn more, see[Aligned Catalog Items: Applications and IT Components](https://help.sap.com/docs/leanix/ea/applications-in-reference-catalog?locale=en-US&state=PRODUCTION&version=CLOUD#loio275c39d17a441014a6b3acadb416005d__aligned_catalog_items_applications_and_it_components).
