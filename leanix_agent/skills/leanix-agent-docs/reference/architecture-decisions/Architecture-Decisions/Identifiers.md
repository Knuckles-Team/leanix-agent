##  Identifiers
Each architecture decision has a unique identifier (ID). This means you can:
  * Easily reference decisions in your communications, which streamlines collaboration and auditing.
  * Track and link decisions in external tools such as Jira or Excel.
  * Maintain a linear, sequential log of decisions that follows best practices.


IDs are automatically generated for architecture decisions based on the prefix defined in the corresponding decision template. For example, decisions created from a template with the prefix "ADR" are numbered ADR-1, ADR-2, and so on. Once generated, IDs remain unchanged to prevent potential duplicates.
Users with permission to manage decision templates can adjust ID prefixes to align them with your organization's naming conventions. To learn more, see [ID Prefixes](https://help.sap.com/docs/leanix/ea/architecture-decision-templates?locale=en-US&state=PRODUCTION&version=CLOUD#loio2758b4647a4410149f75c5924afeff4b__id-prefixes).
