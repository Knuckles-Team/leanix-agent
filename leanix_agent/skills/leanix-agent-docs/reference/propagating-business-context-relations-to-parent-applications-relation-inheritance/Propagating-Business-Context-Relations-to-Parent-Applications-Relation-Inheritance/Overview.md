##  Overview
Business context inheritance ensures that business context relations are automatically propagated from child applications to their parent applications. For example, when a business context is linked to a child application, the same business context should automatically be linked to the parent application. You can automate business context propagation by configuring automations with script actions. By doing so, you can:
  * Reduce manual updates: Eliminate the need to manually add business context relations to parent applications when children are updated.
  * Improve data consistency: Ensure that business context information is always accurate across the application hierarchy.
  * Maintain traceability: Inherited relations are marked with their source, showing which child applications contributed to each business context.
  * Preserve manual relations: Relations manually added to parent applications are not affected by the automation.
