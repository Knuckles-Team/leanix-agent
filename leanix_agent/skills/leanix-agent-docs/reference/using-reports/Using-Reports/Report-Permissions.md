##  Report Permissions
Report permissions for members and viewers are as follows:
Actions | Member | Viewer
---|---|---
Create new report | No | No
Save and Save as options | Only Save as for default reports and shared reports for which the user doesn’t have edit permissions. Once saved in this manner, the report is considered as user's own  | Only Save as for default reports and shared reports for which the user doesn’t have edit permissions. Once saved in this manner, the report is considered as user's own
Save and Save as for shared reports for which the user has edit permissions  | Save and Save as for shared reports for which the user has edit permissions
Save and Save as for own reports | Save and Save as for own reports
Delete a report | Yes: User’s own reports Shared reports for which the user has edit permissions  | Yes: User’s own reports Shared reports for which the user has edit permissions
No: Shared reports for which the user doesn’t have edit permissions Default reports | No: Shared reports for which the user doesn’t have edit permissions Default reports
Share a report | Yes | Yes
Export a report |  Yes However, it depends on whether members have export permissions for all fact sheet types included in the report. By default, the authorization model grants members export permissions for all fact sheet types. For more details, see [Fact Sheet Permissions](https://help.sap.com/docs/leanix/ea/fact-sheet-permissions?locale=en-US&state=PRODUCTION&version=CLOUD "Configure fact sheet permissions for non-admin user roles."). |  No However, it depends on whether viewers have export permissions for all fact sheet types included in the report. By default, viewers do not have export permissions for fact sheets, but the admins can change this. For more details, see [Fact Sheet Permissions](https://help.sap.com/docs/leanix/ea/fact-sheet-permissions?locale=en-US&state=PRODUCTION&version=CLOUD "Configure fact sheet permissions for non-admin user roles.").
Add to collection | No | No
